# 智能养生系统

> 一个算法团队还没招齐、设计交互都没排期、测试用例塞不下、运维积压没清完的项目，被包装成两周上线的战略级智能养生系统 V3.0。

用户把手指放摄像头上测个心率，产品说这是 AI 把脉；手机 CPU 跑满发热，产品说这是元宇宙艾灸；算法没有就先返回 mock 数据，用户不健康就推荐煲汤。核心不是医疗智能化，是所有端一起把“不可能”对齐成“没问题”。

## 技术真相

所谓智能养生系统，本质上是一堆还没具备交付条件的功能需求：

- AI 把脉没有算法，甚至算法团队还在招聘。
- “脉搏数据”实际更像手机摄像头测心率。
- 元宇宙艾灸要求前端做 3D 人体、穴位交互、烟雾特效和温感反馈，温感靠 CPU 占满让手机发热。
- 后端先搭接口壳子，返回 mock 数据，等算法来了再对接。
- 设计稿、交互文档、测试周期、运维排期都没准备好。
- 项目两周上线，解决方案主要是加班和降低问题优先级。

## 领导话术

这不是需求失控，这是面向大健康赛道的快速验证闭环。

- 两周上线不是排期冒进，是抢占 Q2 核心指标窗口期
- 算法团队还没组建，不叫资源缺口，叫预留弹性算法接入层
- mock 数据不是造假，是健康体验保底策略
- 手机发热不是性能问题，是端侧沉浸式温感反馈
- 登录按钮点不了不是阻塞，是验证码路径下的曲线救国
- 每个项目都是 P0，不是没有优先级，是全局战略颗粒度全面拉齐
