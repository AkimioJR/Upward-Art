# 安全底线

> 在局长办公室演示 AI 系统，查 1.8TB 数据库连接超时红屏报错。旭阳把"数据库崩了"翻译成"安全机制拦截了未授权访问"，王处心领神会，当场提炼出"宁断不漏，可信 AI"八个字，转身去找领导把事故讲成案例。顺手安排加个"演示环境功能受限"弹窗——永远的万能保险。

## 技术真相

在局长办公室用单位内网查生产库，命中未优化的查询条件，1.8TB 数据库连接超时，页面白屏转圈两分钟后报红屏错误，满屏英文。本质上就是：换了一个查询条件，系统扛不住了。

## 领导话术

- 不是崩了，是安全机制拦截了未授权访问——安全底线是硬的
- 生产环境和演示环境双隔离架构，领导办公室网络不在授权白名单内，系统自动触发安全拦截
- 这恰恰说明我们的 AI 系统不是一个玩具，它有安全意识，它在保护数据
- 可信不是因为它算得准，是因为它该拦的时候拦得住
- 别人的系统什么都查得出来，不安全；我们的系统该查不了的说查不了，这才安全
- 系统会说"不"——宁可中断演示，也不开放数据访问。这就是可信 AI
- 宁断不漏，可信 AI
