# PPT Wrap Bug / 代码变PPT

## 项目简介

程序员"向上管理话术库"——收集、整理各种把技术问题翻译成领导听得懂（甚至觉得高大上）的语言的案例。

灵感来源于 Bilibili UP主「老头们的快乐生活」(seko阿色) 制作的一系列讽刺短剧，每集都在讲同一个故事：**程序员写三行代码，经过包装翻译，最终变成王处站在台上念的那页 PPT**。

## 核心理念

> 代码本身不重要，变成 PPT 才重要。

每个案例都遵循同一个结构：

```
技术真相 → 包装话术 → 领导认知 → PPT 呈现
```

## 案例结构

cases/                         # 工作区存放案例的目录
├── 001-太极加密/                # 序号加名字（这个名字需要 agent 总结出来）
│   ├── README.md              # 总结文档
│   ├── script.md              # 视频脚本
│   └── source.mp4             # 原始视频


具体内容可以参考模板: template/README.md、template/script.md
可以先复制模板到目标目录再填写

## 使用场景

| 你遇到的问题       | 参考案例             |
| ------------------ | -------------------- |
| 系统出了 bug       | 「自愈系统」话术     |
| 数据不好看         | 「三层指标体系」话术 |
| 要汇报但没技术亮点 | 「区块链审批链」话术 |
| 编码问题被质疑     | 「太极加密法」话术   |

## 背景角色

- **王处（王大川/王大春）**：信息处处长，不懂技术但极擅长包装，能把任何 bug 提炼成哲学体系
- **旭阳**：技术负责人，负责把王处的灵感落地成可演示的东西
- **小孙**：新人程序员，学了点理论但还没学会"翻译"
- **浩宇**：执行层，负责写代码、填日志、画界面

## 免责声明

本项目仅供程序员群体娱乐和自嘲使用。请勿在真实工作中直接套用这些话术。如果你的领导真的被这些话术骗到了，那说明你的领导可能比王处还王处。
