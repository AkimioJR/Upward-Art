# 逐层收口

> 年度考核 26 个指标后天交，系统日均真实业务数据只有 30 条。王处用"体检前准备"类比，教团队把系统日志、鼠标事件、前端加载全部统计进来，凑出 1200 万。旭阳提出"指标树"结构让 26 个数字互相印证，王处提炼出"逐层收口"汇报术——三层封死领导追问：总量让人惊叹、分项让人点头、子项用专业术语让人放弃追问。外加实时跳数大屏和全国辐射影响力地图。

## 技术真相

系统实际日均业务数据：数据库写入约 30 条，查询 400 多次（大部分是王处自己刷新大屏）。为了填 26 个考核指标，团队统计了所有系统产生的日志噪声：页面加载的安全日志（1.2 万次）、后台六层微服务调用（2400 条"数据穿透校验"）、前端鼠标事件（184.32 万次）、AI 语义分析调用等，最终凑到 1198.73 万，约等于 1200 万。

核心技巧"逐层收口"：汇报分三层——第一层总量（1200 万，让领导惊叹）、第二层分项（五个模块拆解，让领导点头）、第三层子项（甩语义分析、向量匹配等专业术语，领导听不懂但不好意思说，就说"不错继续保持"）。三层封死追问。

附加操作：实时跳数大屏（数字从 0 涨到 1200 万后归零重涨）、3D 发光指标树、全国辐射影响力地图（实际只覆盖一个区）。三个往期 bug 被包装成"AI 赋能创新提案"（中西合璧加密、实时穿透保障、数据资产化），已盖章认证。

## 领导话术

- 我不要真实数据——指标不是你日常的状态，是你最好的状态，这叫考前准备
- 隔壁市场监管局日均 2300 万，你信吗？他们跟你们一样，但人家会填
- 上面看的是格子里的数字，数字大你干的多，数字小你干的少。没人会打开数据库数
- 一千两百万，不多不少，比市场监管局少说明我们谦虚，比教育局多说明我们实干
- 汇报不是在讲数字，是在管理领导的追问欲
- 逐层收口：第一层让他惊叹，第二层让他点头，第三层让他放弃追问
- 三层不能少，少了他觉得你敷衍；不能多，多了他觉得你心虚
- 第四层是给审计看的，不是给领导看的
