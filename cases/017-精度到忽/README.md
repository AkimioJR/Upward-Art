# 精度到忽

> 浩宇凌晨赶工用 `double` 存金额，总账 88888.80 后面拖了一串浮点误差尾巴。旭阳不修 bug 反而标红 1247 笔"资金微差异常"，告诉王处"普通系统算到分就瞎了，我们能算到忽——贪腐就是从分以下看不见的零头开始的"。王处当场领悟出"筛子 vs 滤芯"和"戥子称药"的比喻，发明了"防微杜渐廉政预警平台"，要求精度做到"埃"——一粒尘埃那么小的钱都跑不掉。

## 技术真相

浩宇在存储金额字段时使用了 `double` 浮点类型（图省事，比 `BigDecimal` 写着快），导致 100 多万笔数据产生浮点精度误差——总账 88888.80 后面拖了 `0.00000001` 等尾巴。

旭阳拒绝在汇报前修复 bug（来不及重算 100 多万笔，且改对了反而没亮点），而是把浮点误差重新包装：把所有小数点后有多余位数的报销单标红，列表标题写"资金微差异常"。王处分不清浮点误差和真正的资金异常——他只看到系统把分以下的零头揪出来了。

旭阳的后续方案：换成 `BigDecimal` 后面想拖多少零拖多少零，给王处看 20 位精度。"一粒尘埃的钱本来就不存在，我给他的也是一粒尘埃的存在感。"

## 领导话术

- 普通系统算到分就瞎了，分以下厘、毫、丝、忽它全看不见。我们这个系统能算到忽
- 贪腐是从哪开始的？是不是从四舍五入开始的？一笔抹 7 分，一万笔抹多少？700 块
- 普通系统帮着他抹，我们这个系统一忽都不让他抹
- 别人查贪腐是拿个筛子，大窟窿分以上的筛得住，分以下的全漏了。我们这个是滤芯，分厘毫丝忽五层，一忽都漏不过去
- 别的单位查账是拿眼看，差不多就行；我们查账是拿戥子称，一忽都不行
- 上面来检查廉政，人家说账目清楚精确到分；我说我们账目穿透精确到忽
- 精度越高，能揪出来的问题越多——这不是把账算复杂了，是主动发现问题的反腐预警
- 资金监管从分到埃，防微杜渐
